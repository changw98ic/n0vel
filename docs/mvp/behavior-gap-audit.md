# MVP 行为缺口审计

本文档记录当前仍然需要在实现阶段明确或验证的行为型细节。

静态页面与高优先级交互规则已经基本收口；这里保留的是更偏实现和联调层面的剩余问题。

## 当前状态

- 高优先级行为缺口已收口。
- 当前剩余项以实现验证、联调验证和细节体验为主。
- 后续更适合转入代码实现和运行时验证，而不是继续扩写静态状态图。

## 剩余问题

### 阅读器行为

- `Reading Mode` 的键盘翻页规则仍需实装验证：
  - 左右方向键何时翻页
  - 首章 / 终章边界时是否出现轻反馈动效
- 单页阅读态与分页态切换的阈值：
  - 按字符数
  - 按排版高度
  - 按固定分页算法

### 工作台 AI 流

- 多处选区的真正创建方式仍需实现约束：
  - 如何在正文里连续勾选多段
  - 如何避免相邻选区被系统判为重叠
- `前往设置`、`返回正文` 这类 CTA 的真实跳转与焦点恢复：
  - 从阻塞弹窗进入设置后，返回时是否保留原选区
  - 返回正文后是否重新聚焦当前编辑位置

## 当前建议顺序

1. 阅读器键盘翻页与分页阈值
2. 多处选区创建与防重叠约束
3. 阻塞弹窗返回工作台后的焦点恢复
