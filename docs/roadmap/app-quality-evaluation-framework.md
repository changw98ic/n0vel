# 应用质量评估体系 v1

## 评分规则

- **5**: 行业标杆，可作为参考实现
- **4**: 完备且一致，无已知缺陷
- **3**: 核心可用，有可接受的瑕疵
- **2**: 存在影响体验的明显问题
- **1**: 不可用或存在阻断性缺陷
- **0**: 缺失

单项 ≤2 为阻断项，必须修复后才能继续评分。

---

## A. 功能完整性（30%）

### A1 核心链路闭环
| 级别 | 标准 |
|------|------|
| 5 | 每条 PRD 有端到端自动化验证 |
| 4 | 核心链路手工可走通，边界场景有覆盖 |
| 3 | Happy path 通，edge case 有遗漏 |
| 2 | 部分核心链路断裂 |
| 1 | 主流程不可用 |

### A2 功能覆盖率
| 级别 | 标准 |
|------|------|
| 5 | PRD 需求 100% 实现 + 合理的增强 |
| 4 | PRD 需求 100% 实现 |
| 3 | PRD 需求 >90% 实现，缺失非关键 |
| 2 | PRD 需求 >70% 实现 |
| 1 | 核心需求缺失 |

### A3 竞品对等度
| 级别 | 标准 |
|------|------|
| 5 | 核心功能超越同类产品 |
| 4 | 核心功能与同类持平 |
| 3 | 核心功能基本对等，有少量缺失 |
| 2 | 核心功能明显落后 |
| 1 | 缺少竞品必备功能 |

## B. 可靠性（30%）

### B1 数据完整性
| 级别 | 标准 |
|------|------|
| 5 | 写前校验 + 写后验证 + 异常自动恢复 |
| 4 | 写前校验 + 写后验证 |
| 3 | 正常路径数据完整，异常路径有丢失风险 |
| 2 | 存在已知的数据丢失场景 |

### B2 状态一致性
| 级别 | 标准 |
|------|------|
| 5 | 跨 store/跨页面/跨重启状态一致，有自动化验证 |
| 4 | 正常操作状态一致，跨 store 有通知机制 |
| 3 | 大部分一致，偶有延迟刷新 |
| 2 | 编辑后需手动刷新才能看到变化 |
| 1 | 状态经常不同步 |

### B3 异常恢复
| 级别 | 标准 |
|------|------|
| 5 | 崩溃后自动恢复 + 数据不丢失 + 状态可回溯 |
| 4 | 崩溃后数据不丢失，重启可继续 |
| 3 | 崩溃后部分数据可能丢失 |
| 2 | 崩溃后需手动修复 |

> 注：体验是功能的放大器。功能可用但体验差 = 用户不会用。

### C1 操作效率
| 级别 | 标准 |
|------|------|
| 5 | 高频操作 ≤2 步完成 + 键盘快捷键覆盖 |
| 4 | 高频操作 ≤3 步 |
| 3 | 高频操作 ≤5 步 |
| 2 | 高频操作 >5 步 |
| 1 | 操作路径混乱，找不到入口 |

| 级别 | 标准 |
|------|------|

### C4 错误引导
| 级别 | 标准 |
|------|------|
| 5 | 错误消息说明原因 + 给出具体修复路径 |
| 4 | 错误消息说明原因 |

## 评分汇总模板

| 维度 | 权重 | A1 | A2 | A3 | B1 | B2 | B3 | C1 | C2 | C3 | C4 | D1 | D2 | D3 | D4 | E1 | E2 | E3 | F1 | F2 | F3 | G1 | G2 | G3 |
|------|------|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|----|
| 分数 | —    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |    |

**加权总分** = A类均值×0.30 + B类均值×0.30 + C类均值×0.15 + D类均值×0.10 + E类均值×0.05 + F类均值×0.05 + G类均值×0.05

### 总分等级

| 分数区间 | 等级 | 含义 |
|----------|------|------|
| 4.5-5.0 | S | 可发布，行业标杆 |
| 3.5-4.4 | A | 可发布，高质量 |
| 2.5-3.4 | B | 可试用，需打磨 |
| 1.5-2.4 | C | 不可发布，需返工 |
| 0-1.4 | D | 不可用 |

### 阻断规则
- 任一单项 ≤2 → 总分不超过 B
- 任一单项 ≤1 → 总分不超过 C
- 存在数据丢失风险（B1≤2）→ 不可发布
- 存在安全漏洞（F1≤2）→ 不可发布
